🎯 QUICK START
═════════════════════════════════════════════════════════════════════

LOCAL TESTING (2 minutes):
  1. npm install
  2. cp .env.example .env
  3. Edit .env - add BOT_TOKEN from @BotFather
  4. npm install -g pm2
  5. pm2 start ecosystem.config.js
  6. Send /start to your bot on Telegram
  7. pm2 logs → verify response
  8. pm2 delete all

RENDER DEPLOYMENT (10 minutes):
  1. git add . && git commit -m "Deploy" && git push origin main
  2. Render.com → New → Web Service → Select repo
  3. Build from render.yaml (auto-creates 4 services)
  4. Set environment: BOT_TOKEN, WEBHOOK_URL, WEBHOOK_SECRET
  5. Once deployed, run webhook setup (see QUICK_REFERENCE.md)
  6. Test on Telegram → /start should work instantly

1. Test locally with PM2 before Render deployment

🐛 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════

Problem: Bot doesn't respond
→ Check: curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
→ Should show: "pending_update_count": 0
→ Fix: Re-run setWebhook command if pending > 0

Problem: Keep-alive not working
→ Check: Logs for "bot.keep_alive_pong"
→ Should: Appear every 10 minutes
→ Fix: Verify ENABLE_KEEP_ALIVE=true in Render

Problem: Worker keeps restarting
→ Check: Render logs for errors
→ Likely: BOT_TOKEN not set or invalid
→ Fix: Verify environment variables

Problem: Mobile app hangs
→ Check: Response time (should be < 3s)
→ Likely: API timeout or handler not async
→ Fix: Review MOBILE_OPTIMIZATION.md patterns

See DEPLOYMENT_CHECKLIST.md for full troubleshooting.
